异常处理

引擎使用两种异常类型：

  • custom_error — 引擎自定义错误，包含标题和错误信息
  • std::exception — 标准异常（如 SDL 错误）
try { // 引擎操作 } catch (const custom_error& e) { std::cerr << e.title() << ": " << e.what() << std::endl; } catch (const std::exception& e) { std::cerr << e.what() << std::endl; }

常见工作流程

场景 → 精灵 → 组件

  1. 继承 Scene 创建自定义场景
  2. 在 on_enter() 中创建精灵并调用 add_sprite()
  3. 在 on_update() 中更新精灵逻辑
  4. 在 on_render() 中调用基类渲染
  5. 使用 Director 管理场景切换

碰撞检测

  1. 在精灵中保存 CollisionBox* 指针
  2. 在 on_update() 中更新碰撞箱位置
  3. 在 on_enter() 中注册碰撞回调
  4. 碰撞自动由 CollisionManager 处理
  5. 在回调中获取 CollisionInfo

动画

  1. 创建 Atlas 加载纹理序列
  2. 创建 Animation 绑定 Atlas
  3. 将 Animation 添加到 AnimationPlayer
  4. 在 on_update() 中调用 player.on_update()
  5. 在 on_render() 中调用 player.on_render()
